# Remix Markdown Setup

## mdx-plugin and MDX frontmatter support

This article is essentially a test for configuring Markdown in Remix Framework. Markdown, with its user-friendly formatting syntax, can significantly enhance web content creation in the Remix environment.

Visit the following link for the necessary installation steps for setting up the route module for mdx: [Remix setup guide](https://remix.run/docs/en/main/guides/vite#add-mdx-plugin)

Ensure the metadata for the markdown file, such as 'title' and 'description', are appropriately assigned. When testing, setting 'Cache-Control' to 'no-cache' in the header section is recommended. This markdown configuration trial will provide a foundation for more advanced applications within the Remix Framework.


```tsx
// vite.config.ts

import {vitePlugin as remix} from "@remix-run/dev";
import {installGlobals} from "@remix-run/node";
import {defineConfig} from "vite";
import tsconfigPaths from "vite-tsconfig-paths";
import mdx from "@mdx-js/rollup";

installGlobals();

export default defineConfig({
    plugins: [mdx(), remix(), tsconfigPaths()],
});
````
